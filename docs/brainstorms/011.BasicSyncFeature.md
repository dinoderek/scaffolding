Create a basic sync feature that:
1. Can fetch the latest stored state from the backend on first login
2. Periodically saves the local state to the backend 

Out of scope:
* Multi device support 
* Protection against multiple devices 
* Detection of multiple devices 

Journeys: 
* I can see the sync state of the app from the Settings menu (sync enabled, last successful sync)
* When logging in I can enable the sync and download initial state. If I have local state, the local state should be "merged" locally and uploaded to remote.
* If I am logged in I can enable/disable syncing
* If syncing is enabled, the application periodically syncs the state with backend (i.e. it uploads new events - as there is a single device, there is never new state from the backend except at first sync).

Non functional features:
* Connectivity avilability detector
* Frequent delta updates
* Exponential backoff on failures and coalescing of updates into single request (just batching)

UI changes:
* Add sync status, enable and disable to profile.

Future proofing (don't implement now):
* We want to send granular updates to the server to enable low latency social features.
